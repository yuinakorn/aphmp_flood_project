## Register

**product** — เครื่องมือที่รับใช้ workflow ของเจ้าหน้าที่สาธารณสุขหน้าด่าน ไม่ใช่ showcase ของแบรนด์ ดีไซน์ทุกอย่างต้องตอบโจทย์ "งานที่ต้องทำ ณ จุดนั้น" ก่อนความสวย

## วิสัยทัศน์หนึ่งประโยค

ทะเบียนสุขภาพชุมชนเชิงพื้นที่ที่ใช้ได้**ทั้งปี** สำหรับดูแลกลุ่มเปราะบาง และพร้อม**ยกระดับเป็นศูนย์ตอบโต้ภัยพิบัติ**ทันทีเมื่อเกิดน้ำท่วม — บนหน้าจอเดียวกัน ข้อมูลเดียวกัน

## ผู้ใช้และบทบาท (3 ระดับ)

ระบบสลับ "สิทธิ์ใช้งาน" ได้ และ UI/ปุ่ม/ฟอร์มปรับตามบทบาท

| บทบาท | ใคร | งานหลัก |
| --- | --- | --- |
| **อสม. (VHV)** | อาสาสมัครสาธารณสุขประจำหมู่บ้าน | บันทึกภาคสนาม: เยี่ยมประเมินสุขภาพ, คัดกรองสภาพจิตใจ (MCAT), ยื่นคำร้องขอช่วยเหลือ — mobile-first, low-connectivity |
| **รพ.สต. / โรงพยาบาล (officer)** | เจ้าหน้าที่สั่งการระดับตำบล/อำเภอ | คัดกรองความเร่งด่วน (Triage), จัดสรรทีมกู้ภัยให้คำร้อง, ปิดเคส, ดูแลทะเบียนวันปกติ |
| **ผู้บัญชาการ EOC (admin)** | ผู้บริหารศูนย์สั่งการ | ภาพรวมทั้งหมด, ขึ้นทะเบียน/แบ่งโซนทีมกู้ภัย, ตัดสินใจเชิงยุทธศาสตร์ |

(แมปกับ `UserRole` เดิมในโค้ดภายหลัง — บทบาทอื่นถือเป็น subset ของ 3 ระดับนี้)

## สองโหมดของระบบ

ระบบมี state กลางหนึ่งตัว: **Normal ↔ Crisis** สลับที่ header

- **โหมดปกติ (ดูแลสุขภาพ)** — โทนสงบ เน้นทะเบียน การเยี่ยมสะสม ความครบถ้วนของข้อมูล ใช้ 360 วันต่อปี
- **โหมดวิกฤต (น้ำท่วม)** — เปิดเมื่อมี **Incident** ที่กำลังเกิด (หรือ telemetry น้ำเกินเกณฑ์ → auto-trigger) เปลี่ยนเป็นโทนเร่งด่วน, เปิดแถบประกาศภัย, เปิดระบบจัดลำดับคำร้อง, ข้อมูลภาคสนามใหม่ผูกกับ incident อัตโนมัติ

โหมดวิกฤต **เปิดทับ**โหมดปกติ ไม่ใช่ระบบแยก — ข้อมูล/ทะเบียนชุดเดียวกัน

## เสาหลักของผลิตภัณฑ์ (Product pillars)

1. **ทะเบียนกลุ่มเปราะบาง/ครัวเรือน** — หลักการ "หมุด = บ้าน": หนึ่งหมุดแสดงสมาชิกทุกคนในครัวเรือน + เบอร์ทุกคน, ระดับความเปราะบาง (ติดเตียง/สูงอายุ/พิการ/ตั้งครรภ์), **อุปกรณ์พยุงชีพที่ต้องนำติดตัว** (ออกซิเจน, น้ำยาล้างไต, ยาต้านชัก), ผู้ดูแล/อสม.รับผิดชอบ
2. **ปฏิบัติการภาคสนาม (อสม.)** — แบบประเมินสุขภาพ (ชีพจร/อาการ), คัดกรองสภาพจิตใจ→ส่ง MCAT, แผนยา/อุปกรณ์ (พอใช้กี่วัน), ยื่นคำร้องขอช่วยเหลือพร้อมพิกัด
3. **ศูนย์จัดการคำร้อง (Command Hub)** — **Triage ทางคลินิก** แดง(1-2 ชม.)/เหลือง(เฝ้าระวัง)/เขียว(เตรียมแผน), จัดสรรทีมให้คำร้อง, ติดตามสถานะ Pending→Processing→Completed
4. **ประสานทีมกู้ภัย (Rescue coordination)** — ขึ้นทะเบียนหน่วย (เรือ, GMC, EMS, MCAT, ครัว) + **แบ่งโซนรับผิดชอบ** เพื่อกันการเข้าช่วยซ้ำซ้อน/บ้านที่ถูกทิ้งร้าง (บทเรียนภาคสนามจริง)
5. **ระดับน้ำ + GIS น้ำท่วม** — โทรมาตร/จุดวัด (น่าน N64, แม่สาย) แสดงระดับ→กระตุ้นเกณฑ์เตือน, overlay โซนน้ำท่วมบนแผนที่ตามความรุนแรง
6. **ศูนย์พักพิง** — ความจุ/occupancy/ความพร้อมด้านสุขภาพ (รองรับติดเตียง/ออกซิเจน)

## บุคลิกแบรนด์ (Brand personality)

**สะอาด-เป็นมืออาชีพ มีสีสันที่มีความหมาย** (clean-professional, semantically colorful)

- พื้นสว่าง การ์ดชัด เงานุ่ม — อ่านง่ายกลางวัน ใช้ในสำนักงาน รพ.สต. ได้สบาย
- สีทำงานเป็น "ภาษา": แดง=วิกฤต, เหลือง/ส้ม=เฝ้าระวัง/ต้องการช่วย, เขียว=ปลอดภัย, น้ำเงิน=อพยพแล้ว/ข้อมูล, คราม=โครงสร้างพื้นฐาน
- ไอคอนคมชุดเดียว (Lucide) — **ไม่ใช้ emoji เป็นไอคอน** (mockup ใช้ emoji เพื่อ prototype เร็ว; ของจริงใช้ icon set)
- โหมดปกติ = สงบ; โหมดวิกฤต = เร่งด่วนชัดเจนแต่ไม่ตื่นตระหนก (severity สื่อด้วยตำแหน่ง+น้ำหนัก+สี ไม่ใช่ไฟกะพริบทั้งจอ)

## Anti-references (อย่าให้หน้าตาเป็นแบบนี้)

1. **เว็บราชการไทย** — กรอบเยอะ, clip-art, ฟอนต์ปนกัน, emoji-as-icon
2. **SaaS dashboard ม่วง-gradient ทั่วไป** — หลีกเลี่ยง gradient บนตัวอักษร, ไม่ใช้ม่วงเป็นสีหลัก (ม่วง/คราม = โครงสร้างพื้นฐานเท่านั้น)
3. **UI วิกฤตที่ตื่นตระหนก** — แดงทั้งจอกะพริบ; แดงสงวนให้ "วิกฤต/ในเขตน้ำท่วม" เท่านั้น
4. **หน้าตา prototype** — inline style, spacing มั่ว, emoji icon

> เปลี่ยนจาก legacy: เดิมห้าม card grid และห้ามสีเยอะ — เวอร์ชันนี้ **รับ card + สีเชิงความหมาย** เป็นหัวใจ แต่ยังคุมวินัย (ทุก spacing มาจาก scale เดียว, ทุกสีมี role)

## หลักการออกแบบเชิงกลยุทธ์

1. **หน้าจอเดียว ตอบคำถามเดียวที่สำคัญ** — EOC dashboard ตอบ "น้ำอยู่ไหน + ใครเสี่ยง + ทีมไหนทำอะไรแล้ว"
2. **สถานะมาก่อน chrome** — แถบ stats (เปราะบางทั้งหมด/ต้องการช่วย/ปลอดภัย/ทีมกู้ภัย) อยู่บนสุดของ panel ขวา
3. **เปิดเผยเท่าที่จำเป็น (PDPA)** — ข้อมูลตัวบุคคล/อุปกรณ์เปิดหลัง sign-in + บันทึก audit log ทุกครั้ง
4. **จบที่การลงมือ (Action over information)** — ทุก detail view จบด้วยปุ่มกริยา: โทรหาผู้ดูแล, บันทึกเยี่ยม, ขอ EMS, จัดทีม, ทำเครื่องหมายอพยพแล้ว
5. **ทวิภาษาไร้ภาษีซ้ำซ้อน** — ไทยเป็นหลัก, เลข/เวลาเป็น Latin/mono อยู่ร่วมกันได้
6. **ปรับตามบทบาทและโหมด** — UI ชุดเดียว แต่ปุ่ม/ฟอร์ม/สีปรับตาม role + mode

## โครงสร้างหน้าจอ (Hybrid)

- **EOC Dashboard (ใหม่)** — หน้ารวมแบบ mockup: ซ้าย = แผนที่ GIS + จำลองระดับน้ำ, ขวา = stats + 3 แท็บ (ทะเบียนเปราะบาง / ศูนย์จัดการคำร้อง / กู้ภัยและหน่วยเคลื่อนที่เร็ว). ตอบภาพรวม + สั่งการเร็ว
- **หน้า admin เชิงลึก (คงไว้ restyle)** — family-folder, vulnerable, infra, water-level, incidents สำหรับงานจัดการข้อมูลละเอียดที่ dashboard ไม่เหมาะ
- **แผนที่ปฏิบัติการเต็มจอ** — คงไว้สำหรับเฝ้าระวัง real-time

## Accessibility & PDPA

- WCAG 2.2 AA สำหรับ contrast ทุกการแสดงข้อมูล; AAA บน risk pill วิกฤต/เฝ้าระวัง/ปลอดภัย
- สถานะสื่อด้วย **ไอคอน + ข้อความ + ตำแหน่ง** ไม่พึ่งสีอย่างเดียว (เจ้าหน้าที่บางคนตาบอดสี)
- เคารพ reduced-motion: pulse/bounce ยุบเป็น opacity
- ข้อมูลสุขภาพส่วนบุคคลตาม PDPA — เปิดเฉพาะผู้ได้รับอนุญาต, ทุกการดู/แก้ไขถูกบันทึก

## Out of scope (v1)

- ไม่มี public registration; admin เป็นคนเปิดสิทธิ์
- ไม่มีโมดูล logistics อาหาร/น้ำเต็มรูป (เก็บเฉพาะที่กระทบความพร้อมด้านสุขภาพของศูนย์พักพิง)
- ไม่มี incident command ดับเพลิง/กู้ภัยเต็มรูปเกินกว่าที่จำเป็นต่อการดูแลกลุ่มเปราะบาง
- ไม่มีฟีเจอร์ social/sharing
