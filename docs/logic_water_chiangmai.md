# การพัฒนาระบบ Alert และวิเคราะห์ระดับน้ำเชียงใหม่ (P.1 และ P.67)

สรุปแนวคิดแบบ “เอาไปใช้จริงในศูนย์บัญชาการ/ระบบ Alert” โดยอิงจากข้อมูลในเว็บ + เหตุการณ์น้ำท่วมเชียงใหม่จริง + threshold ที่ใช้อ้างอิง

## 1. หลักคิดการเตือนภัย: Threshold ต้องมี 2 ชั้น

ในการเฝ้าระวัง อย่าดูแค่ **"ระดับน้ำ (Level)"** อย่างเดียว ให้ดูควบคู่กัน 2 ปัจจัยคือ:
1. **ระดับน้ำปัจจุบัน (Absolute level)**
2. **ความเร็วในการเพิ่มขึ้น (Rate of rise)**

> **ตัวอย่าง:** น้ำเพิ่มจาก 1.8m → 2.4m ใน 4 ชั่วโมง (ถึงระดับน้ำจะยังไม่วิกฤต แต่ "ขึ้นเร็วผิดปกติ" ต้องเตือนก่อน)

---

### ลุ่มน้ำปิง: P.67 → P.1 (เชียงใหม่)

จากหน้าเว็บมีค่าอ้างอิงของสถานีดังนี้:

| สถานี | จุดที่ตั้ง | Threshold ในเว็บ |
| :--- | :--- | :--- |
| **P.67** | บ้านแม่แต | 3.00 ม. |
| **P.1** | สะพานนวรัฐ | 3.70 ม. |

*(ตัวอย่างสถานการณ์วันที่ 15 พ.ค. 2026: P.67 ≈ 0.58 ม., P.1 ≈ 1.57 ม. ถือว่าปลอดภัยมาก)*

## 2. Threshold ที่แนะนำ (Operational)

### P.67 บ้านแม่แต (ต้นน้ำเชียงใหม่)
**หน้าที่:** ใช้เป็น Early Warning ก่อนมวลน้ำเดินทางเข้าสู่ตัวเมือง

| สัญลักษณ์ | ระดับเตือนภัย | ระดับน้ำ | Action |
| :---: | :--- | :--- | :--- |
| 🟢 | ปกติ | `< 2.0m` | monitor ทุก 1–3 ชม. |
| 🟡 | เฝ้าระวัง | `2.0–2.5m` | แจ้งศูนย์, monitor รายชั่วโมง |
| 🟠 | เตรียมพร้อม | `2.5–2.8m` | แจ้ง อปท. / โรงพยาบาล / พื้นที่เสี่ยง |
| 🔴 | วิกฤต | `>= 2.8m` | activate response |
| 🚨 | อันตรายสูง | `>= 3.0m` | เสี่ยงน้ำหลากเข้าเมือง |

> **เหตุผล:** เว็บตั้ง reference ไว้ที่ 3.0 ม. และในเหตุการณ์ปี 2567 ใช้ช่วง ~2.95m เป็นตัว trigger เฝ้าระวังจริง

### P.1 สะพานนวรัฐ (ตัวเมืองเชียงใหม่)
**หน้าที่:** ใช้ตัดสินใจเชิงปฏิบัติการในเขตเมือง

| สัญลักษณ์ | ระดับเตือนภัย | ระดับน้ำ | Action |
| :---: | :--- | :--- | :--- |
| 🟢 | ปกติ | `< 2.5m` | monitor |
| 🟡 | เริ่มสูง | `2.5–3.0m` | แจ้งเฝ้าระวัง |
| 🟠 | น้ำเริ่มมาก | `3.0–3.4m` | เตรียม response |
| 🔴 | ใกล้วิกฤต | `3.4–3.7m` | เปิด EOC |
| 🚨 | ใกล้ล้นตลิ่ง | `>= 3.7m` | ปฏิบัติการเต็มรูปแบบ |

## 3. Rule เรื่อง “น้ำขึ้นเร็ว” (Rate of Rise)
กฏข้อนี้บางครั้งสำคัญและเห็นผลก่อน Threshold ระดับน้ำเสียอีก

### Trigger 1 — ขึ้นเร็วผิดปกติ
**เงื่อนไข:** น้ำเพิ่มระดับ `> 0.20m` ภายใน 3 ชั่วโมง
- **ตัวอย่าง:** `10:00 = 2.1m` ➡️ `13:00 = 2.35m`
- **Action:** แม้ระดับน้ำจะยังไม่ถึง Threshold วิกฤต ก็ต้องส่ง Alert ทันที

### Trigger 2 — Upstream สูงกว่า Downstream มาก
**เงื่อนไข:** สถานีต้นน้ำเพิ่มระดับรวดเร็ว แต่สถานีในเมืองระดับยังต่ำ (แปลว่ามวลน้ำกำลังเดินทางเข้าเมือง)
- **ตัวอย่าง:** `P.67 = 2.7m`, `P.1 = 2.4m`
- **Action:** ควรเตรียมการรับมือมวลน้ำในตัวเมืองล่วงหน้า

## 4. System Implementation

### Config ที่แนะนำ (พร้อมนำไปใช้ในระบบ)

```json
{
  "stations": {
    "P.67": {
      "name": "บ้านแม่แต",
      "warning": 2.0,
      "prepare": 2.5,
      "critical": 2.8,
      "danger": 3.0,
      "rapidRise": 0.2
    },
    "P.1": {
      "name": "สะพานนวรัฐ",
      "warning": 2.5,
      "prepare": 3.0,
      "critical": 3.4,
      "danger": 3.7,
      "rapidRise": 0.2
    }
  }
}
```

### Logic Alert (Pseudo-code)

```javascript
function getWaterAlert(level, prevLevel, threshold) {
  const rise = level - prevLevel;

  if (level >= threshold.danger) return "danger";
  if (level >= threshold.critical) return "critical";
  if (rise >= threshold.rapidRise) return "rapid-rise";
  if (level >= threshold.prepare) return "prepare";
  if (level >= threshold.warning) return "warning";
  
  return "normal";
}
```

## 📊 Threshold และช่วงวิกฤตรายสถานี (ปี 2566–2567)

------

จากประกาศของชลประทานเชียงใหม่ปี 2567:

สถานี P.1 สะพานนวรัฐ
ประมาณ 3.7 ม. = ใกล้ตลิ่ง / เฝ้าระวังสูง
สูงกว่าตลิ่ง = เริ่มเสี่ยงล้นตลิ่งและท่วมพื้นที่ลุ่มต่ำ
สถานี P.67 บ้านแม่แต
ประมาณ 2.95–3.0 ม. ถูกเรียกว่า “เริ่มวิกฤต” ในช่วงน้ำหลากใหญ่ปี 2567

ถ้าสรุปแบบใช้ได้จริงสายศูนย์บัญชาการ/DevOps monitoring ผมแนะนำ “rule of thumb” แบบนี้:


| ระดับ       | ควรทำอะไร                                                   |
| ----------- | ----------------------------------------------------------- |
| ปกติ        | ต่ำกว่า ~60–70% ของระดับวิกฤต → monitor ปกติ                |
| เฝ้าระวัง   | >70% ของระดับวิกฤต → เปิด dashboard เฝ้าดูรายชั่วโมง        |
| เตรียมพร้อม | >85–90% → แจ้ง อปท./โรงพยาบาล/จุดเสี่ยง                     |
| วิกฤต       | ใกล้ตลิ่ง หรือแนวโน้มเพิ่มเร็ว → activate incident response |



สำหรับ P.1 เชียงใหม่ ถ้าเอาจากเหตุการณ์จริง:

<2.5 ม. → ปกติ
2.5–3.2 ม. → เริ่มจับตา
3.2–3.7 ม. → เฝ้าระวังจริงจัง
≥3.7 ม. → ใกล้วิกฤต / เสี่ยงน้ำเอ่อพื้นที่ลุ่มต่ำ
