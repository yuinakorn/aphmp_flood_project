จากข้อมูลตารางรายชื่อผู้ใช้บริการ ศูนย์พักพิงชั่วคราว วัดพรหมวิหาร (รพ.แม่สาย) ที่เพิ่มเข้ามา สามารถอธิบายรายละเอียดและวัตถุประสงค์ของการมีตารางเหล่านี้ได้ดังนี้ครับ
ตารางศูนย์พักพิงประกอบด้วยอะไรบ้าง?
ตารางนี้ถูกแบ่งออกเป็น 3 ส่วนหลัก (3 แท็บข้อมูล) ได้แก่:
กลุ่มผู้ป่วยติดเตียงที่ต้องส่งต่อ รพ.แม่สาย
กลุ่มผู้พักพิงทั่วไปของวัดพรหมวิหาร
กลุ่มผู้พักพิงโซนอาคารชาติพันธุ์
โดยโครงสร้างภายในของแต่ละตารางจะประกอบด้วย 2 ส่วนสำคัญ คือ:
ส่วนสรุปยอดรวม (Summary Dashboard): สรุปจำนวนตัวเลขแบบ Real-time เช่น จำนวนผู้เข้าพักสะสม, จำนวนผู้เข้าพักปัจจุบัน, จำนวนที่ย้ายออกแล้ว, และจำนวนผู้ป่วยที่ต้อง Admit ไปโรงพยาบาล
ส่วนข้อมูลรายบุคคล (Individual Records): มีการเก็บข้อมูลอย่างละเอียด ได้แก่
ข้อมูลระบุตัวตนและติดต่อ: ชื่อ-สกุล, เลขบัตรประชาชน, วันเกิด, อายุ, เพศ, สัญชาติ, ที่อยู่ (เลขที่, หมู่, ตำบล) และเบอร์โทรศัพท์
ข้อมูลด้านสุขภาพ: โรคประจำตัว, ประวัติแพ้อาหาร, และประวัติแพ้ยา
ข้อมูลสถานะการพักพิง: จุดรับเข้า/สถานที่พักพิง, วันที่เข้าพัก, วันที่ย้ายออก, สถานะปัจจุบัน (พักอยู่/ย้ายออก), และหน่วยงานกู้ภัยที่นำส่ง

ตารางนี้มีไว้ทำไม? (วัตถุประสงค์ในการใช้งาน)
1. เพื่อติดตามและยืนยันสถานะการอพยพ (Tracking & Verification) จากปัญหาที่ทีมเชียงรายพบในการปฏิบัติงานจริงคือ เมื่อเกิดน้ำท่วม การตามหาว่าผู้ป่วยติดเตียงหรือกลุ่มเปราะบางถูกอพยพออกมาแล้วหรือยัง เป็นเรื่องที่ใช้พลังงานมากและตรวจสอบยาก
 ตารางนี้จึงมีไว้เพื่อใช้เช็คชื่อ (Register) ว่ากลุ่มเปราะบางได้รับการช่วยเหลือออกจากพื้นที่เสี่ยงแล้ว และรู้สถานะชัดเจนว่าปัจจุบันพักอยู่ที่ศูนย์ หรือถูกส่งต่อไป รพ. แล้ว
2. เพื่อบริหารจัดการทางการแพทย์และแจกจ่ายทรัพยากร การมีช่อง "โรคประจำตัว แพ้ยา แพ้อาหาร" มีความสำคัญมากในภาวะวิกฤต เช่น หากทราบว่าผู้พักพิงมีโรคประจำตัวที่ต้องรับการฟอกไต (CKD) ทีมแพทย์จะสามารถเตรียมแผนขนส่งน้ำยาล้างไต (PD) มาที่ศูนย์พักพิงได้ถูกต้อง
 หรือสามารถจัดเตรียมยาและเวชภัณฑ์ได้ตรงกับความต้องการของแต่ละบุคคลโดยไม่เกิดอันตรายจากการแพ้ยา
3. เพื่อนำไปจัดทำรายงานสถานการณ์ (Situation Report) ข้อมูลที่ถูกกรอกในส่วนสรุปยอดรวมของตารางนี้ จะถูกนำไปใช้เป็นฐานข้อมูลในการทำรายงานภาพรวมรายวัน (ตัวอย่างเช่น ภาพรายงานสถานการณ์ของ รพ.แม่สาย ที่ระบุชัดเจนว่ามี "ศูนย์พักพิง 1 แห่ง คือ วัดพรหมวิหาร") เพื่อให้ผู้บริหารหรือ EOC (ศูนย์ปฏิบัติการส่วนหน้า) ใช้ประกอบการตัดสินใจสั่งการ
4. เป็นแหล่งข้อมูลป้อนเข้าสู่ระบบ Health GIS (Data Input) ในมุมของการพัฒนาระบบ Health GIS ที่เรากำลังทำอยู่ ตารางนี้คือหน้าตาของข้อมูล กลุ่ม B (Incident Data & Person Status Log) ที่หน้างานใช้จริง การเก็บวันเวลาเข้า-ออก และสถานะ (พักอยู่/ย้ายออก) คือการทำ State Machine (อพยพแล้ว → ปลอดภัย) เพื่อป้องกันผู้ป่วยตกหล่นระหว่างเกิดภัยพิบัตินั่นเองครับ
